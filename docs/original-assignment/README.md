# Original Assignment Summary

This directory intentionally contains a redacted summary instead of the copied application-system attachment.

The Exa take-home asked for a small applet that demonstrates how Exa can be a force multiplier for financial-services companies adopting AI agents. The submission should use Exa APIs against real external content, make the end user and workflow clear, and include a deployed app, a short demo video, and a source repository.

Key prompt themes:

- Identify challenges financial-services teams face when adopting AI agents.
- Show what external signals or data Exa can capture that others cannot.
- Explain what market or GTM wedge those signals unlock for Exa.
- Anticipate barriers in the sales process and how the applet helps overcome them.
- Build either an internal enablement tool for AEs/FDEs or an external workflow applet.

This project chose the internal enablement route: source-backed FSI agent-adoption signals for Exa AEs, with Exa Answer-generated prospect Webset suggestions and branded demo Webset pages.
